# 演習1：HRSのドメイン分析

## 1. 目的

HRSの問題領域を分析し、主要な概念と概念間の関係をUMLクラス図として表す。
また、具体的な状況をUMLオブジェクト図として表し、クラス図が妥当であるか確認する。

## 2. 対象システム

対象はホテル予約システム（HRS）である。
利用者は部屋を予約し、予約番号を受け取る。
チェックイン時には、利用者がホテル受付係に予約番号を伝え、受付係は部屋番号を伝える。
チェックアウト時には、受付係が宿泊料を伝え、利用者が宿泊料を支払う。

## 3. 抽出した主な概念

- 利用者
- 予約
- 部屋
- ホテル
- ホテル受付係
- 宿泊
- 宿泊料
- 支払い

## 4. UMLクラス図

PlantUMLファイル：

[クラス図](../diagrams/class_diagram.puml)

このクラス図では、利用者、予約、部屋、ホテル受付係、宿泊、宿泊料、支払いなどをHRSの主要な概念として表した。
利用者は予約を行い、予約は部屋を対象とする。
また、予約はチェックイン後の宿泊につながり、宿泊には宿泊料が発生する。

具体的なオブジェクトを作成することで、クラス図の関係が実際の場面でも自然に使えることを確認した。

## 7. 修正点

レビューの結果、予約と部屋の関係に多重度を追加した。
また、宿泊料と支払いの関係を追加し、チェックアウト時の流れが分かるようにした。
