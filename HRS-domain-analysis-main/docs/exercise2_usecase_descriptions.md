# 演習2：ユースケース記述

## 1. ユースケース：空室を確認して部屋を予約する

### アクタ

- 利用者

### 事前条件

- 利用者が宿泊したい日付を決めている。
- ホテルに予約可能な部屋が登録されている。

### 事後条件

- 利用者の予約が作成される。
- 利用者に予約番号が通知される。

### 基本系列

1. 利用者はチェックイン日、チェックアウト日、人数を指定する。
2. HRSは指定された条件に合う空室を検索する。
3. HRSは利用可能な部屋の一覧を表示する。
4. 利用者は予約したい部屋を選択する。
5. HRSは予約を作成する。
6. HRSは予約番号を利用者に通知する。

### 代替・例外系列

- 条件に合う空室がない場合、HRSは空室がないことを利用者に通知する。
- 利用者が部屋を選択しない場合、HRSは予約を作成しない。

## 2. ユースケース：予約番号でチェックインする

### アクタ

- 利用者
- ホテル受付係

### 事前条件

- 利用者が予約番号を持っている。
- 予約が有効である。

### 事後条件

- 利用者のチェックインが完了する。
- 利用者に部屋番号が伝えられる。

### 基本系列

1. 利用者はホテル受付係に予約番号を伝える。
2. ホテル受付係はHRSに予約番号を入力する。
3. HRSは予約情報を検索する。
4. HRSは予約内容を表示する。
5. ホテル受付係は予約内容を確認する。
6. HRSはチェックイン状態に更新する。
7. ホテル受付係は利用者に部屋番号を伝える。

### 代替・例外系列

- 予約番号が存在しない場合、HRSは予約が見つからないことを表示する。
- 予約日が当日でない場合、HRSはチェックインできないことを表示する。

## 3. ユースケース：宿泊料を精算してチェックアウトする

### アクタ

- 利用者
- ホテル受付係

### 事前条件

- 利用者がチェックイン済みである。
- 宿泊料を計算できる状態である。

### 事後条件

- 宿泊料の支払いが完了する。
- 利用者のチェックアウトが完了する。
- 部屋の状態が利用可能に戻る。

### 基本系列

1. 利用者はホテル受付係にチェックアウトを申し出る。
2. ホテル受付係はHRSに部屋番号または予約番号を入力する。
3. HRSは宿泊情報を検索する。
4. HRSは宿泊料を計算する。
5. HRSは宿泊料を表示する。
6. ホテル受付係は利用者に宿泊料を伝える。
7. 利用者は宿泊料を支払う。
8. ホテル受付係は支払い完了をHRSに入力する。
9. HRSはチェックアウト状態に更新する。
10. HRSは部屋の状態を利用可能に更新する。

### 代替・例外系列

- 宿泊情報が見つからない場合、HRSは宿泊情報がないことを表示する。
- 支払いが完了しない場合、HRSはチェックアウトを完了しない。
